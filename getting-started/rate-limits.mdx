---
title: "Rate limits"
description: "Default API rate limits for akta.pro plans and endpoints"
---

Rate limits depend on your active plan. Higher-tier plans include higher request throughput, while Enterprise plans can be configured with custom limits based on committed usage, endpoint requirements, and deployment needs.

The following table shows the default rate limits by plan:

| Plan | Default rate limit | Applies to |
| :-- | :-- | :-- |
| Pay as you go | 100 requests / minute | All API requests |
| Subscription | 200 requests / minute | All API requests |
| Enterprise | Custom | Custom endpoints, feeds, and agreed usage limits |

<Note>
  Rate limits apply to API requests made against Akta endpoints. Enterprise customers can request custom limits based on expected volume, endpoint configuration, and production requirements.
</Note>

## What counts as a request?

Each request to the News, Company Enrichment and Alternative Data endpoint counts toward your plan's rate limit.

<Info>
  Usage for all endpoints is charged in credits. Rate limits control request throughput; credit usage controls billing.
</Info>

## Pricing

Rate limits are separate from pricing. For cost information, see [Pricing](/credits).

## Need higher limits?

If you need higher throughput, custom endpoints, de-duplicated feeds, or committed-volume pricing, contact [sales@akta.pro](mailto:sales@wokelo.ai) with your expected usage volume and requirements
