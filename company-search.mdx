## Quick start

The Company Search API helps you resolve a company input into one or more matching company records. You can pass a company name, website, or company UUID using the `company` query parameter.

Each result includes the company identifier, company name, website, product category, and IPO status. Use the returned `company_id` when calling downstream company data APIs such as Company Enrichment.

Results are returned synchronously in the HTTP response.

<Note>
  This is a synchronous API. The results are returned directly in the HTTP response.
</Note>

## Endpoint

Method: GET

`/v1/companies/search`

## Base URL

`https://api.akta.pro`

## Authentication

Pass your API key in the `x-api-key` request header.

`-H "x-api-key: YOUR_API_KEY"`

<Warning>
  Do not expose production API keys in client-side code, public repositories, screenshots, or shared documentation. The examples below use a placeholder key.
</Warning>

## Sample request

<CodeGroup>

```bash cURL
curl -G "https://api.akta.pro/v1/companies/search" \
  -H "x-api-key: YOUR_API_KEY" \
  --data-urlencode "company=canva"
```

```python Python
import requests

url = "https://api.akta.pro/v1/companies/search"
headers = {"x-api-key": "YOUR_API_KEY"}
params = {
    "company": "canva",
}

response = requests.get(url, headers=headers, params=params)
response.raise_for_status()

payload = response.json()

for company in payload["data"]:
    print(company["company_id"])
    print(company["name"])
    print(company["website"])
    print(company["product_category"])
    print(company["ipo_status"])
    print()
```

```javascript JavaScript
const params = new URLSearchParams({
  company: "canva",
});

const response = await fetch(
  `https://api.akta.pro/v1/companies/search?${params}`,
  { headers: { "x-api-key": "YOUR_API_KEY" } }
);

if (!response.ok) {
  throw new Error(`Request failed with status ${response.status}`);
}

const payload = await response.json();

for (const company of payload.data) {
  console.log(company.company_id);
  console.log(company.name);
  console.log(company.website);
  console.log(company.product_category);
  console.log(company.ipo_status);
}
```

</CodeGroup>

## Request reference

### Query parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `company` | string | Yes | Company name, website, or company UUID to resolve. Example: `canva.com`. |

## Response

A successful response returns a list of matching companies and pagination metadata.

```json
{
  "data": [
    {
      "company_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
      "name": "Acme Corporation",
      "website": "https://acme.com",
      "product_category": "SaaS",
      "ipo_status": "public"
    }
  ],
  "pagination": {
    "total": 42,
    "limit": 10,
    "offset": 0
  }
}
```

### Top-level response fields

| Field | Type | Description |
| --- | --- | --- |
| `data` | array | Array of matching company objects. |
| `pagination` | object | Pagination metadata for the returned result set. |

### Company object fields

| Field | Type | Description |
| --- | --- | --- |
| `company_id` | string | Unique identifier for the company. UUID v4 format. |
| `name` | string | Legal or registered company name. |
| `website` | string | Primary website URL. |
| `product_category` | string | Primary product or service category. Example: `SaaS`, `FinTech`. |
| `ipo_status` | string | Public market status. Example: `public`, `private`, `acquired`, `unknown`. |

### Pagination object fields

| Field | Type | Description |
| --- | --- | --- |
| `total` | integer | Total number of matching companies. |
| `limit` | integer | Maximum number of companies returned in the response. |
| `offset` | integer | Number of companies skipped before the current response. |

## Error responses

### 400 Bad Request

Returned when the request contains invalid or conflicting input.

```json
{
  "error": "INVALID_INPUT",
  "message": "Invalid company search input."
}
```

### 422 Unprocessable Entity

Returned when the required `company` parameter is missing.

```json
{
  "error": "MISSING_PARAMETER",
  "message": "The company query parameter is required."
}
```

### 500 Internal Server Error

Returned when an unexpected server error occurs.

```json
{
  "error": "SERVER_ERROR",
  "message": "An unexpected error occurred. Please try again."
}
```

## Implementation notes

- Use the `company` parameter to pass a company name, website, or company UUID.
- Use the returned `company_id` as the preferred identifier for downstream APIs.
- Store `company_id`, `name`, `website`, `product_category`, and `ipo_status` when building company lookup or enrichment workflows.
- If multiple companies are returned, use `website` and `name` to confirm the correct match before enriching the company.
- Do not expose API keys in browser code or public client-side applications.
