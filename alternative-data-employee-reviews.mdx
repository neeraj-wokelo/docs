## 1. Overview

The Employee Reviews API returns aggregated workplace sentiment data and individual employee reviews for any company. A single request returns the overall company rating, eight dimension-level scores (culture, work-life balance, compensation, senior management, diversity and inclusion, business outlook, CEO approval, and recommendation rate), and a paginated list of individual reviews ŌĆö each with the reviewer's pros, cons, job title, tenure, employment status, location, and per-dimension ratings.

This is a **synchronous GET API** ŌĆö pass a company identifier and optional pagination parameters and receive the full response immediately.

<Info>
  This API is synchronous. Results are returned directly in the HTTP response ŌĆö no job submission or polling required.
</Info>

## 5. Response

### Response envelope

```json
{
  "data": {
    "uuid": "00000jw-canva",
    "employee_reviews": {
      "overall_rating": { ... },
      "other_ratings":  [ ... ],
      "reviews":        { ... }
    }
  },
  "limit": 10,
  "offset": 0,
  "credits_consumed": 1.5
}
```

| Field | Type | Description |
| --- | --- | --- |
| `data` | object | Container object with the company `uuid` and an `employee_reviews` object. |
| `data.uuid` | string | Akta UUID for the queried company. |
| `data.employee_reviews` | object | Contains `overall_rating`, `other_ratings`, and `reviews`. |
| `limit` | integer | The `limit` value applied to this response. |
| `offset` | integer | The `offset` value applied to this response. |
| `credits_consumed` | float | Credits consumed by this request. |

### `overall_rating`

A dict keyed by review provider. Values are floats on a 0ŌĆō5 scale:

```json
{
  "overall_rating": {
    "glassdoor": 3.9
  }
}
```

### `other_ratings`

An array of dimension rating objects, one per dimension per provider:

```json
{
  "other_ratings": [
    { "type": "business_outlook",        "provider": "glassdoor", "value": 0.7  },
    { "type": "ceo",                     "provider": "glassdoor", "value": 0.84 },
    { "type": "recommend_to_friend",     "provider": "glassdoor", "value": 0.69 },
    { "type": "compensation_and_benefits","provider": "glassdoor", "value": 3.9 },
    { "type": "culture_and_values",      "provider": "glassdoor", "value": 3.7  },
    { "type": "diversity_and_inclusion", "provider": "glassdoor", "value": 4.0  },
    { "type": "senior_management",       "provider": "glassdoor", "value": 3.1  },
    { "type": "work_life_balance",       "provider": "glassdoor", "value": 3.7  }
  ]
}
```

| Field | Type | Description |
| --- | --- | --- |
| `type` | string | Rating dimension name. See the dimension scale table below. |
| `provider` | string | Review provider identifier (e.g. `"glassdoor"`). |
| `value` | float | Rating value. Scale depends on `type` ŌĆö see below. |

<Warning>
  The `value` field uses **two different scales** depending on the dimension type. `business_outlook`, `ceo`, and `recommend_to_friend` are **approval ratios between 0.0 and 1.0** (e.g. `0.84` = 84%). All other dimensions are **scores on a 0ŌĆō5 scale**. Never display raw values without applying the correct scale.
</Warning>

**Dimension scale reference:**

| `type` | Scale | How to display |
| --- | --- | --- |
| `business_outlook` | 0.0 ŌĆō 1.0 | Multiply by 100 ŌåÆ percentage (e.g. `0.7` ŌåÆ `"70%"`) |
| `ceo` | 0.0 ŌĆō 1.0 | Multiply by 100 ŌåÆ percentage (e.g. `0.84` ŌåÆ `"84% CEO approval"`) |
| `recommend_to_friend` | 0.0 ŌĆō 1.0 | Multiply by 100 ŌåÆ percentage (e.g. `0.69` ŌåÆ `"69% would recommend"`) |
| `compensation_and_benefits` | 0.0 ŌĆō 5.0 | Display as star score (e.g. `3.9` ŌåÆ `"3.9/5.0"`) |
| `culture_and_values` | 0.0 ŌĆō 5.0 | Display as star score |
| `diversity_and_inclusion` | 0.0 ŌĆō 5.0 | Display as star score |
| `senior_management` | 0.0 ŌĆō 5.0 | Display as star score |
| `work_life_balance` | 0.0 ŌĆō 5.0 | Display as star score |

### Paginating through all reviews

Retrieve the full review corpus for downstream analysis.

```python
import requests

HEADERS  = {"x-api-key": "<YOUR_API_KEY>"}
COMPANY  = "canva.com"
PAGE_SIZE = 100

all_reviews = []
offset      = 0

while True:
    r = requests.get(
        "https://api.akta.pro/api/v1/company/employee-reviews",
        headers=HEADERS,
        params={"company": COMPANY, "limit": PAGE_SIZE, "offset": offset}
    )
    data    = r.json()
    emp     = data["data"]["employee_reviews"]
    batch   = emp["reviews"]["data"]
    total   = emp["reviews"]["total"]

    if not batch:
        break

    all_reviews.extend(batch)
    print(f"Fetched {len(all_reviews)} / {total} reviews...")

    if len(all_reviews) >= total:
        break

    offset += PAGE_SIZE

print(f"\nTotal reviews retrieved: {len(all_reviews)}")

# Separate current vs former employees
current = [r for r in all_reviews if r["is_current_employee"]]
former  = [r for r in all_reviews if not r["is_current_employee"]]
print(f"Current employees: {len(current)}  |  Former: {len(former)}")
```

## 7. Error Handling

| Status | Meaning | Cause & Resolution |
| --- | --- | --- |
| `200 OK` | Success | Data returned successfully. |
| `400 Bad Request` | Invalid parameters | Missing `company` parameter, invalid value, or malformed request. Check the `message` field. |
| `401 Unauthorized` | Auth failed | Missing or invalid `x-api-key` header. Verify your API key. |
| `403 Forbidden` | Insufficient access | Your plan does not include access to this endpoint. Contact [support@akta.pro](mailto:support@akta.pro). |
| `404 Not Found` | Company not found | The `company` value could not be resolved. Verify the domain or UUID using the [Company Search API](/akta-company-search-doc). |
| `429 Too Many Requests` | Rate limit exceeded | Implement exponential back-off. The response includes a `Retry-After` header. |
| `500 Internal Server Error` | Server error | Retry after a brief delay. If the issue persists, contact [support@akta.pro](mailto:support@akta.pro). |

**Retry with exponential back-off:**

```python
import time, requests

def fetch_with_retry(params, api_key, max_retries=3):
    headers = {"x-api-key": api_key}
    for attempt in range(max_retries):
        try:
            r = requests.get(
                "https://api.akta.pro/api/v1/company/employee-reviews",
                headers=headers,
                params=params,
                timeout=30
            )
            if r.status_code == 429:
                time.sleep(2 ** attempt)
                continue
            r.raise_for_status()
            return r.json()
        except requests.exceptions.Timeout:
            if attempt == max_retries - 1:
                raise
            time.sleep(1)
    raise Exception(f"Failed after {max_retries} attempts")
```

## 8. Best Practices

**`other_ratings` uses two scales ŌĆö always check the dimension type before displaying**

Three dimensions (`business_outlook`, `ceo`, `recommend_to_friend`) are approval ratios between 0.0 and 1.0. The remaining five are scores on a 0ŌĆō5 scale. Build a lookup dict and apply the correct formatting for each type:

```python
RATIO_DIMS = {"business_outlook", "ceo", "recommend_to_friend"}

dim_lookup = {r["type"]: r["value"] for r in emp["other_ratings"]}

for dim, value in dim_lookup.items():
    if dim in RATIO_DIMS:
        display = f"{value * 100:.0f}%"        # ratio ŌåÆ percentage
    else:
        display = f"{value:.1f} / 5.0"         # score ŌåÆ star scale
    print(f"  {dim}: {display}")
```

**`ceo_rating` on individual reviews is a string enum, not a float**

The aggregate `ceo` entry in `other_ratings` is a 0ŌĆō1 float. The `ceo_rating` field on each individual review object is a string (`"APPROVE"`, `"DISAPPROVE"`, or `null`). These fields measure the same concept but have completely different types ŌĆö never compare or mix them:

```python
# Aggregate ŌĆö float ratio
ceo_approval_pct = dim_lookup.get("ceo", 0) * 100         # e.g. 84.0%

# Per-review ŌĆö string enum
ceo_sentiment = review.get("ceo_rating")                   # "APPROVE", "DISAPPROVE", or None
```

**Guard all per-dimension review ratings against `null`**

All seven per-review dimension fields (`career_opportunities_rating`, `ceo_rating`, `compensation_and_benefits_rating`, etc.) can be `null` when the reviewer did not provide a score. Always check before accessing:

```python
# ŌØī TypeError when rating is null
avg = sum(r["culture_and_values_rating"] for r in reviews) / len(reviews)

# Ō£ģ Filter nulls first
rated = [r["culture_and_values_rating"] for r in reviews if r.get("culture_and_values_rating") is not None]
avg   = sum(rated) / len(rated) if rated else None
```

**`review_date` and `location` can be empty strings ŌĆö not null**

Unlike most APIs where absent values are `null`, these fields return an empty string `""` when not provided. Use truthiness checks rather than `is not None`:

```python
# ŌØī Won't catch empty strings
if review["review_date"] is not None:
    ...

# Ō£ģ Correct ŌĆö empty string is falsy
if review.get("review_date"):
    ...
```

**Use `limit=0` or `limit=1` when you only need aggregate scores**

The `overall_rating` and `other_ratings` aggregate fields are always returned regardless of `limit`. When you only need the overview scores for a benchmark or monitoring check, set `limit=1` to minimise response size and credit consumption:

```python
# Aggregate scores only ŌĆö minimal reviews payload
r = requests.get(url, headers=HEADERS, params={"company": "canva.com", "limit": 1})
emp = r.json()["data"]["employee_reviews"]
# overall_rating and other_ratings are always present
```

**Use `is_current_employee` and `years_of_employment` to segment reviews**

Separating current from former employees, or filtering by tenure, can surface significantly different sentiment patterns ŌĆö particularly useful when assessing post-acquisition or post-layoff cultural impact:

```python
current_reviews = [r for r in reviews if r["is_current_employee"]]
veteran_reviews = [r for r in reviews if r["years_of_employment"] >= 3]
```

**Use `helpful_count` to surface the most credible reviews**

Reviews with a higher `helpful_count` have been up-voted by the community and tend to be more detailed and representative. Sort by `helpful_count` descending for qualitative diligence or executive briefings:

```python
top_reviews = sorted(reviews, key=lambda r: r.get("helpful_count", 0), reverse=True)
```
