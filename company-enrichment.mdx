# Company Enrichment

> Enrich a company with structured company intelligence including firmographics, go-to-market details, headcount, website traffic, funding, financials, acquisitions, and investments.

## Quick start

The Company Enrichment API returns structured company intelligence for a given company. Pass a company website or company UUID in the `company` field.

Use the optional `sections` field to control which enrichment sections are returned. If `sections` is omitted, all available sections are returned.

Results are returned synchronously in the HTTP response.

<Note>
  Company UUIDs can be obtained from the Company Search API. Use the UUID when you want the most precise company match.
</Note>

## Endpoint

Method: POST

`/v1/companies/enrich`

## Base URL

`https://api.akta.pro`

## Authentication

Pass your API key in the `x-api-key` request header.

```bash
-H "x-api-key: YOUR_API_KEY"
```

<Warning>
  Do not expose production API keys in client-side code, public repositories, screenshots, or shared documentation. The examples below use a placeholder key.
</Warning>

## Sample request

<CodeGroup>

```bash cURL
curl "https://api.akta.pro/v1/companies/enrich" \
  -X POST \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "company": "https://www.canva.com",
    "sections": ["firmographics", "funding"]
  }'
```

```python Python
import requests

url = "https://api.akta.pro/v1/companies/enrich"
headers = {
    "x-api-key": "YOUR_API_KEY",
    "Content-Type": "application/json",
}
payload = {
    "company": "https://www.canva.com",
    "sections": ["firmographics", "funding"],
}

response = requests.post(url, headers=headers, json=payload)
response.raise_for_status()

payload = response.json()

firmographics = payload["data"].get("firmographics")
funding = payload["data"].get("funding")

if firmographics:
    print(firmographics["name"])
    print(firmographics["website"])
    print(firmographics["location"])
    print(firmographics["founded"])
    print(firmographics["type"])
    print(firmographics["operating_status"])
    print(firmographics["ticker"])

if funding:
    overall = funding["overall"]
    print(overall["funding_stage"])
    print(overall["total_funding"])
    print(overall["last_funding_date"])
    print(overall["num_funding_rounds"])
```

## Request reference

### Body parameters

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `company` | string | Yes | Company website or company UUID. Example: `https://www.canva.com` or `3fa85f64-5717-4562-b3fc-2c963f66afa6`. |
| `sections` | array of strings | No | Sections to return. If omitted, all available sections are returned. |

## Available sections

| Value | Description |
| --- | --- |
| `firmographics` | Name, location, founded year, company type, operating status, and ticker. |
| `gtm_and_business_model` | Go-to-market strategy and business model details. |
| `headcount` | Employee count data. |
| `website_traffic` | Website traffic metrics. |
| `funding` | Funding stage, total raised, last round date, and number of funding rounds. |
| `public_company_financials` | Financial data for public companies. |
| `uk_private_company_financials` | Financial data for UK private companies. |
| `acquisitions` | Company acquisition activity. |
| `investments` | Company investment activity. Available on enterprise tier. |

## Response

A successful response returns a status and a `data` object. The keys inside `data` match the requested sections. Only requested sections are present.

```json
{
  "status": "success",
  "data": {
    "firmographics": {
      "name": "Canva",
      "website": "http://www.canva.com",
      "location": "Sydney, Australia",
      "founded": 2013,
      "type": "private",
      "operating_status": "Operating",
      "ticker": null
    },
    "funding": {
      "overall": {
        "funding_stage": "Secondary Market",
        "total_funding": 2508450682.0,
        "last_funding_date": "2025-08-20",
        "num_funding_rounds": "21"
      }
    }
  }
}
```

### Top-level response fields

| Field | Type | Description |
| --- | --- | --- |
| `status` | string | Request status. Returns `success` or `error`. |
| `data` | object | Object containing the requested enrichment sections. |

### Firmographics fields

| Field | Type | Description |
| --- | --- | --- |
| `name` | string | Company name. |
| `website` | string | Primary company website. |
| `location` | string | Company location. Example: `Sydney, Australia`. |
| `founded` | integer | Year the company was founded. |
| `type` | string | Company type. Example: `public`, `private`, `acquired`, `unknown`. |
| `operating_status` | string | Operating status. Example: `Operating`, `Closed`. |
| `ticker` | string or null | Stock ticker. Returns `null` if the company is private. |

### Funding fields

| Field | Type | Description |
| --- | --- | --- |
| `overall.funding_stage` | string | Latest funding stage. Example: `Series B`, `Secondary Market`. |
| `overall.total_funding` | float | Total funding raised in USD. |
| `overall.last_funding_date` | string | ISO 8601 date of the latest funding round. |
| `overall.num_funding_rounds` | string | Total number of funding rounds. |

## Error responses

### 400 Bad Request

Returned when the request contains invalid input.

```json
{
  "status": "error",
  "error": "INVALID_INPUT",
  "message": "Invalid company enrichment input."
}
```

### 422 Unprocessable Entity

Returned when the required company identifier is missing.

```json
{
  "status": "error",
  "error": "MISSING_PARAMETER",
  "message": "The company field is required."
}
```

### 404 Company Not Found

Returned when no company matches the provided input.

```json
{
  "status": "error",
  "error": "COMPANY_NOT_FOUND",
  "message": "No company found for the provided identifier."
}
```

### 500 Internal Server Error

Returned when an unexpected server error occurs.

```json
{
  "status": "error",
  "error": "SERVER_ERROR",
  "message": "An unexpected error occurred. Please try again."
}
```

## Implementation notes

- Use the Company Search API first when you need to resolve a company name into a canonical company UUID.
- Pass a website or company UUID in the `company` field.
- Use `sections` to request only the enrichment sections needed for your workflow.
- Omit `sections` when you want all available sections.
- Use `firmographics` for company identity and basic profile data.
- Use `funding` for investment research, market mapping, and private-company intelligence workflows.
- The `investments` section is available only on enterprise tier.
- Store only the sections required by your downstream workflow to reduce payload size and simplify processing.
