## Quick start

The Product Reviews API returns product review data for a given company. Pass a company website or company UUID in the `company` query parameter.

Each result represents a reviewed product and includes the product name, average rating, star distribution, and a list of review summaries.

Results are returned synchronously in the HTTP response.

<Note>
  Company UUIDs can be obtained from the Company Search API. Use the UUID when you want the most precise company match.
</Note>

## Endpoint

Method: GET

`/v1/companies/reviews`

## Base URL

`https://api.akta.pro`

## Authentication

Pass your API key in the `x-api-key` request header.

```bash
-H "x-api-key: YOUR_API_KEY"
```

<Warning>
  Do not expose production API keys in client-side code, public repositories, screenshots, or shared documentation. The examples below use a placeholder key.
</Warning>

## Sample request

<CodeGroup>

```bash cURL
curl -G "https://api.akta.pro/v1/companies/reviews" \
  -H "x-api-key: YOUR_API_KEY" \
  --data-urlencode "company=canva.com" \
  --data-urlencode "limit=10" \
  --data-urlencode "offset=0"
```

```python Python
import requests

url = "https://api.akta.pro/v1/companies/reviews"
headers = {"x-api-key": "YOUR_API_KEY"}
params = {
    "company": "canva.com",
    "limit": 10,
    "offset": 0,
}

response = requests.get(url, headers=headers, params=params)
response.raise_for_status()

payload = response.json()

for product in payload["data"]:
    print(product["product_name"])
    print(product["rating"])
    print(product["star_distribution"])

    for review in product["reviews"]:
        print(review["review_id"])
        print(review["review_title"])
    print()
```

```javascript JavaScript

}
```

## Request reference

### Query parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `company` | string | Yes | Company website or company UUID. Example: `canva.com` or `3fa85f64-5717-4562-b3fc-2c963f66afa6`. |
| `limit` | integer | No | Maximum number of reviews to return per product. Default: `10`. Max: `100`. |
| `offset` | integer | No | Number of results to skip. Default: `0`. |
| `product_filtering` | string | No | Product filtering parameter used to return reviews for selected products. |

## Response

A successful response returns a status, an array of product review objects, and pagination metadata.

```json
{
  "status": "success",
  "data": [
    {
      "product_name": "Canva Pro",
      "rating": 4.6,
      "star_distribution": {
        "5": 1200,
        "4": 430,
        "3": 95,
        "2": 30,
        "1": 18
      },
      "reviews": [
        {
          "review_id": "a1b2c3d4-e5f6-7890-ab12-cd34ef567890",
          "review_title": "Best design tool for non-designers"
        },
        {
          "review_id": "b2c3d4e5-f6a7-8901-bc23-de45fa678901",
          "review_title": "Great value but limited exports on free plan"
        }
      ]
    }
  ],
  "pagination": {
    "total": 42,
    "limit": 10,
    "offset": 0
  }
}
```

### Top-level response fields

| Field | Type | Description |
| --- | --- | --- |
| `status` | string | Request status. Returns `success` or `error`. |
| `data` | array | Array of product review objects. |
| `pagination` | object | Pagination metadata for the returned result set. |

### Product review object fields

| Field | Type | Description |
| --- | --- | --- |
| `product_name` | string | Name of the reviewed product. |
| `rating` | float | Overall average rating. Example: `4.5`. |
| `star_distribution` | object | Count of reviews by star rating. Keys are `"1"` through `"5"`. |
| `reviews` | array | Array of individual review summary objects. |

### Review object fields

| Field | Type | Description |
| --- | --- | --- |
| `review_id` | string | Unique identifier for the review. |
| `review_title` | string | Title of the review. |

### Pagination object fields

| Field | Type | Description |
| --- | --- | --- |
| `total` | integer | Total number of matching results. |
| `limit` | integer | Maximum number of reviews returned per product. |
| `offset` | integer | Number of results skipped before the current response. |

## Error responses

### 401 Unauthorized

Returned when the API key is missing or invalid.

```json
{
  "status": "error",
  "error": "UNAUTHORIZED",
  "message": "Invalid or missing x-api-key."
}
```

### 400 Bad Request

Returned when the request contains invalid input.

```json
{
  "status": "error",
  "error": "INVALID_INPUT",
  "message": "Invalid product reviews input."
}
```

### 422 Unprocessable Entity

Returned when the required company identifier is missing.

```json
{
  "status": "error",
  "error": "MISSING_PARAMETER",
  "message": "The company query parameter is required."
}
```

### 404 Company Not Found

Returned when no company matches the provided input.

```json
{
  "status": "error",
  "error": "COMPANY_NOT_FOUND",
  "message": "No company found for the provided identifier."
}
```

### 500 Internal Server Error

Returned when an unexpected server error occurs.

```json
{
  "status": "error",
  "error": "SERVER_ERROR",
  "message": "An unexpected error occurred. Please try again."
}
```

## Pagination

Use `limit` and `offset` together to paginate through matching reviews.

```text
Page 1: limit=10&offset=0
Page 2: limit=10&offset=10
Page 3: limit=10&offset=20
```

<Note>
  The response includes pagination metadata. Use `total` to understand the number of matching results and `offset` to request the next page.
</Note>

## Implementation notes

- Use the Company Search API first when you need to resolve a company name into a canonical company UUID.
- Pass a website or company UUID in the `company` query parameter.
- Use `limit` to control the number of reviews returned per product.
- Use `offset` with `limit` for pagination.
- Use `product_filtering` when you need reviews for selected products only.
- Store `product_name`, `rating`, `star_distribution`, `review_id`, and `review_title` for most product review workflows.
- Do not expose API keys in browser code or public client-side applications.
