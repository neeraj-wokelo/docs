## 1. Overview

The Akta Company Enrichment API returns structured data for any company from Akta's native database. Pass a company website domain or UUID and specify which sections you need — only the requested sections are returned, keeping responses lean and processing fast.

This is a **synchronous GET API** — submit a JSON body and receive enriched company data immediately in the HTTP response. No polling required.

**The available sections:**

| Section | What you get |
| --- | --- |
| `firmographics` | Name, website, location, founded year, company type, operating status, and stock ticker |
| `funding` | Funding stage, total capital raised, last round date, and number of rounds |
| `headcount` | Employee count and headcount data |
| `website_traffic` | Web traffic metrics and visitor estimates |
| `gtm_and_business_model` | Go-to-market strategy and business model details |
| `public_company_financials` | Revenue, margins, and valuation data — public companies only |
| `uk_private_company_financials` | Income statement and balance sheet data for UK-registered private companies |
| `acquisitions` | M&A history — companies acquired and deal details |
| `investments` | Portfolio investments made by the company _(enterprise tier only)_ |

**Common use cases:**

- **CRM enrichment on demand** — call the API from a webhook or automation to instantly populate Salesforce, HubSpot, or DealCloud records when a new company is added
- **Deal screening data pull** — request `firmographics` \+ `funding` \+ `acquisitions` for a shortlist of targets to build a fast pre-IC data pack
- **Funding intelligence** — pull `funding` for a set of companies to map raise history, identify stage, and track total capital deployed
- **Traffic and growth signals** — combine `headcount` and `website_traffic` to proxy revenue trajectory and hiring velocity for private companies
- **UK private company financials** — use `uk_private_company_financials` to pull Companies House data for UK-based private targets where Crunchbase data is thin

<Info>
  This API is synchronous. Enriched data is returned directly in the HTTP response — no job submission or polling required.
</Info>

## 2. Quick Start

**Step 1 — Make a simple request**

<CodeGroup>

```bash cURL
curl --location 'https://api.akta.ai/v1/company/enrichment' \
  --header 'x-api-key: <YOUR_API_KEY>' \
  --header 'Content-Type: application/json' \
  --data '{
    "company": "canva.com",
    "sections": ["firmographics", "funding"]
  }'
```

```python Python
import requests

response = requests.post(
    "https://api.akta.ai/v1/company/enrichment",
    headers={
        "x-api-key":    "<YOUR_API_KEY>",
        "Content-Type": "application/json"
    },
    json={
        "company":  "canva.com",
        "sections": ["firmographics", "funding"]
    }
)
data = response.json()["data"]
print(f"{data['firmographics']['name']} — {data['firmographics']['company_status']}")
print(f"Total funding: ${data['funding']['overall']['total_funding'] / 1e9:.1f}B")
```

</CodeGroup>

```python
response = requests.post(
```

## 4. Request Reference

**Endpoint**

```text
POST https://api.akta.ai/v1/company/enrichment
```

All parameters are passed as JSON in the request body.

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `company` | string | **Required** | Company identifier. Accepts either a **website domain** (e.g. `"canva.com"` or `"https://www.canva.com"`) or a **company UUID** (e.g. `"3fa85f64-5717-4562-b3fc-2c963f66afa6"`). UUIDs are returned by the [Company Search API](#). |
| `sections` | string\[\] | Optional | Array of section names to include in the response. Returns all available sections if omitted. Request only what you need — each additional section adds to response size and processing time. See the available sections table in Section 1. |

<Info>
  `sections` defaults to all available sections when omitted. For time-sensitive integrations, always specify the sections you need rather than relying on the default — this reduces response size significantly and avoids receiving large sections like `acquisitions` when only `firmographics` is needed.
</Info>

**Full request example:**

## 5. Response

### Response envelope

```json
{
  "status": "success",
  "data": {
    "firmographics": { ... },
    "funding": { ... }
  }
}
```

| Field | Type | Description |
| --- | --- | --- |
| `status` | string | `"success"` when data was returned successfully. |
| `data` | object | Dict keyed by section name. Only the sections you requested are present. |

### Section schemas

**`firmographics`**

```json
{
  "firmographics": {
    "name": "Canva",
    "website": "https://www.canva.com",
    "location": "Sydney, Australia",
    "founded": 2013,
    "company_status": "private",
    "operating_status": "Operating",
    "ticker": null
  }
}
```

| Field | Type | Description |
| --- | --- | --- |
| `name` | string | Company display name. |
| `website` | string | Primary website URL. |
| `location` | string | City and country of headquarters (e.g. `"Sydney, Australia"`). |
| `founded` | integer | Year the company was founded. |
| `company_status` | string | Ownership structure: `"public"`, `"private"`, `"acquired"`, or `"unknown"`. |
| `operating_status` | string | Current operational state: `"Operating"`, `"Closed"`, or similar. |
| `ticker` | string or null | Stock ticker symbol (e.g. `"NASDAQ:FRSH"`). `null` for private companies. |

**`funding`**

```json
{
  "funding": {
    "overall": {
      "funding_stage": "Secondary Market",
      "total_funding": 2508450682.0,
      "last_funding_date": "2025-08-20",
      "num_funding_rounds": "21"
    }
  }
}
```

The `funding` section contains an `overall` object with aggregate funding summary fields:

| Field | Type | Description |
| --- | --- | --- |
| `overall.funding_stage` | string | Most recent round type (e.g. `"Series B"`, `"Secondary Market"`, `"Public"`). |
| `overall.total_funding` | float | Total capital raised in raw USD (e.g. `2508450682.0` = \$2.51B). |
| `overall.last_funding_date` | string | Date of the most recent funding event in `YYYY-MM-DD` format. |
| `overall.num_funding_rounds` | string | Total number of funding rounds as a string (e.g. `"21"`). Parse with `int()` before arithmetic. |

**`headcount`, `website_traffic`, `gtm_and_business_model`, `public_company_financials`, `uk_private_company_financials`, `acquisitions`, `investments`**

These sections follow the same envelope pattern — each section key maps to a structured object with relevant fields. Refer to the [full schema reference](#) for complete field-level documentation on all sections.

<Info>
  `public_company_financials` is populated only for publicly listed companies. `uk_private_company_financials` is populated only for UK-registered private entities. `investments` is available on enterprise tier only. All other sections are available on all plans.
</Info>

### CRM enrichment helper — resolve then enrich

Resolve a company name via Company Search, then enrich synchronously for a CRM record update.

```python
import requests

HEADERS = {
    "x-api-key":    "<YOUR_API_KEY>",
    "Content-Type": "application/json"
}

def enrich_by_name(company_name, sections):
    # Step 1: resolve to UUID via Company Search
    search = requests.get(
        "https://api.akta.ai/v1/company/search",
        headers={"x-api-key": "<YOUR_API_KEY>"},
        params={"query": company_name}
    )
    results = search.json().get("data", [])
    if not results:
        raise ValueError(f"Company not found: {company_name}")

    company_id = results[0]["company_id"]

    # Step 2: enrich using the UUID
    enrich = requests.post(
        "https://api.akta.ai/v1/company/enrichment",
        headers=HEADERS,
        json={"company": company_id, "sections": sections}
    )
    return enrich.json()["data"]

data = enrich_by_name("Stripe", ["firmographics", "funding"])
overall = data["funding"]["overall"]
print(f"{data['firmographics']['name']}: ${overall['total_funding'] / 1e9:.1f}B raised ({overall['funding_stage']})")
```

### Batch enrichment across a watchlist

Enrich multiple companies concurrently by running parallel requests.

```python
import requests, concurrent.futures

HEADERS = {
    "x-api-key":    "<YOUR_API_KEY>",
    "Content-Type": "application/json"
}

WATCHLIST = ["stripe.com", "figma.com", "notion.so", "loom.com", "miro.com"]
SECTIONS  = ["firmographics", "funding", "headcount"]

def enrich_company(domain):
    r = requests.post(
        "https://api.akta.ai/v1/company/enrichment",
        headers=HEADERS,
        json={"company": domain, "sections": SECTIONS}
    )
    return domain, r.json().get("data", {})

with concurrent.futures.ThreadPoolExecutor(max_workers=5) as pool:
    results = dict(pool.map(lambda d: enrich_company(d), WATCHLIST))

print(f"{'Company':<20} {'Status':<12} {'Stage':<22} {'Total Raised'}")
print("-" * 72)
for domain, data in results.items():
    firm    = data.get("firmographics", {})
    funding = data.get("funding", {}).get("overall", {})
    total   = funding.get("total_funding", 0)
    total_s = f"${total / 1e9:.1f}B" if total >= 1e9 else (f"${total / 1e6:.0f}M" if total > 0 else "—")
    print(
        f"{firm.get('name', domain):<20} "
        f"{firm.get('company_status', '—'):<12} "
        f"{funding.get('funding_stage', '—'):<22} "
        f"{total_s}"
    )
```

## 7. Error Handling

The API uses standard HTTP status codes. All errors are returned synchronously.

| Status | Meaning | Cause & Resolution |
| --- | --- | --- |
| `200 OK` | Success | Enriched data returned in response body. |
| `400 Bad Request` | Invalid parameters | Missing `company` field, unrecognised section name, or malformed UUID. Check the `message` field for details. |
| `401 Unauthorized` | Auth failed | Missing or invalid `x-api-key` header. Verify your API key. |
| `403 Forbidden` | Insufficient access | Your plan does not include access to a requested section (e.g. `investments` on a non-enterprise plan). Contact [support@akta.ai](mailto:support@akta.ai). |
| `404 Not Found` | Company not found | The `company` value could not be resolved to a known company. Verify the domain or UUID using the [Company Search API](#). |
| `429 Too Many Requests` | Rate limit exceeded | Implement exponential back-off. The response includes a `Retry-After` header. |
| `500 Internal Server Error` | Server error | Retry after a brief delay. If the issue persists, contact [support@akta.ai](mailto:support@akta.ai). |

**Error response example:**

```json
{
  "status": "error",
  "error": "COMPANY_NOT_FOUND",
  "message": "No company found matching the provided identifier."
}
```

**Retry with exponential back-off:**

```python
import time, requests

def enrich_with_retry(body, api_key, max_retries=3):
    headers = {
        "x-api-key":    api_key,
        "Content-Type": "application/json"
    }
    for attempt in range(max_retries):
        try:
            r = requests.post(
                "https://api.akta.ai/v1/company/enrichment",
                headers=headers,
                json=body,
                timeout=30
            )
            if r.status_code == 429:
                time.sleep(2 ** attempt)
                continue
            r.raise_for_status()
            return r.json()["data"]
        except requests.exceptions.Timeout:
            if attempt == max_retries - 1:
                raise
            time.sleep(1)
    raise Exception(f"Failed after {max_retries} attempts")
```

## 8. Best Practices

**Always specify `sections` — never rely on the default**

When `sections` is omitted, all available sections are returned. For large sections like `acquisitions` or `public_company_financials`, this increases response size significantly with no benefit if you only need `firmographics`. Always be explicit:

```python
# ❌ Returns everything — slow and large
json={"company": "canva.com"}

# ✅ Returns only what you need
json={"company": "canva.com", "sections": ["firmographics", "funding"]}
```

**Accept both domain and full URL formats**

The `company` field accepts both bare domains (`"canva.com"`) and full URLs (`"https://www.canva.com"`). Either format works — use whichever is more natural for your pipeline. If you have a UUID from Company Search, prefer that for the most reliable resolution:

```python
# All three are valid
json={"company": "canva.com"}
json={"company": "https://www.canva.com"}
json={"company": "3fa85f64-5717-4562-b3fc-2c963f66afa6"}
```

**`num_funding_rounds` is a string — parse before arithmetic**

The `funding.overall.num_funding_rounds` field returns a string (e.g. `"21"`), not an integer. Parse it before any numeric operations:

```python
num_rounds = int(data["funding"]["overall"]["num_funding_rounds"])
```

**`total_funding` is raw USD — scale before displaying**

The `total_funding` field is a raw float in USD (e.g. `2508450682.0`). Always divide by `1e6` or `1e9` before presenting to users:

```python
total_m = data["funding"]["overall"]["total_funding"] / 1e6   # in millions
total_b = data["funding"]["overall"]["total_funding"] / 1e9   # in billions
label = f"${total_b:.1f}B" if total_b >= 1 else f"${total_m:.0f}M"
```

**Use Company Search to resolve names to UUIDs before enriching**

If your input is a company name (not a domain), resolve it to a UUID via the Company Search API first. Passing a display name like `"Canva"` directly as the `company` field will not resolve. Use the domain or UUID returned by Company Search:

```python
search = requests.get(
    "https://api.akta.ai/v1/company/search",
    headers={"x-api-key": API_KEY},
    params={"query": "Canva"}
)
company_id = search.json()["data"][0]["company_id"]
```

**Check `company_status` before requesting financial sections**

`public_company_financials` only returns data for publicly listed companies, and `uk_private_company_financials` only for UK-registered private entities. Requesting these sections for companies that don't qualify returns empty data. Check `firmographics.company_status` first to avoid unnecessary requests:

```python
firm = data["firmographics"]

if firm["company_status"] == "public":
    # Request public financials
    pass
elif firm["location"].endswith("United Kingdom"):
    # May have UK private financials
    pass
```

**`investments` requires enterprise tier — handle `403` gracefully**

The `investments` section is gated to enterprise plans. If your plan does not include it, requesting it returns a `403 Forbidden`. Handle this in your section selection logic rather than letting it surface as an uncaught error:

```python
try:
    data = enrich_company("canva.com", ["firmographics", "investments"])
except requests.HTTPError as e:
    if e.response.status_code == 403:
        # Re-request without the gated section
        data = enrich_company("canva.com", ["firmographics"])
```
