## Quick start

The Employee Reviews API returns employee review metrics for a given company. Pass a company website or company UUID in the `company` query parameter.

Each result includes an `overview` object with overall employee rating, business outlook, CEO approval, recommendation score, compensation and benefits rating, culture and values rating, diversity and inclusion rating, senior management rating, and work-life balance rating.

Results are returned synchronously in the HTTP response.

<Note>
  Company UUIDs can be obtained from the Company Search API. Use the UUID when you want the most precise company match.
</Note>

## Endpoint

Method: GET

`/v1/companies/employee-reviews`

## Base URL

`https://api.akta.pro`

## Authentication

Pass your API key in the `x-api-key` request header.

`-H "x-api-key: YOUR_API_KEY"`

<Warning>
  Do not expose production API keys in client-side code, public repositories, screenshots, or shared documentation. The examples below use a placeholder key.
</Warning>

## Sample request

<CodeGroup>

```bash cURL
curl -G "https://api.akta.pro/v1/companies/employee-reviews" \
  -H "x-api-key: YOUR_API_KEY" \
  --data-urlencode "company=canva.com" \
  --data-urlencode "limit=10" \
  --data-urlencode "offset=0"
```

```python Python
import requests

url = "https://api.akta.pro/v1/companies/employee-reviews"
headers = {"x-api-key": "YOUR_API_KEY"}
params = {
    "company": "canva.com",
    "limit": 10,
    "offset": 0,
}

response = requests.get(url, headers=headers, params=params)
response.raise_for_status()

payload = response.json()

for review in payload["data"]:
    overview = review["overview"]
    print(overview["rating"])
    print(overview["business_outlook_rating"])
    print(overview["ceo_rating"])
    print(overview["recommend_to_friend_rating"])
    print(overview["compensation_and_benefits_rating"])
    print(overview["culture_and_values_rating"])
    print(overview["diversity_and_inclusion_rating"])
    print(overview["senior_management_rating"])
    print(overview["work_life_balance_rating"])
    print()
```

```javascript JavaScript
const params = new URLSearchParams({
  company: "canva.com",
  limit: "10",
  offset: "0",
});

const response = await fetch(
  `https://api.akta.pro/v1/companies/employee-reviews?${params}`,
  { headers: { "x-api-key": "YOUR_API_KEY" } }

  throw new Error(`Request failed with status ${response.status}`);
}

for (const review of payload.data) {

  console.log(overview.rating);
  console.log(overview.business_outlook_rating);
  console.log(overview.ceo_rating);
  console.log(overview.recommend_to_friend_rating);
  console.log(overview.compensation_and_benefits_rating);
  console.log(overview.culture_and_values_rating);
  console.log(overview.diversity_and_inclusion_rating);
  console.log(overview.senior_management_rating);
  console.log(overview.work_life_balance_rating);
}
```

## Request reference

### Query parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `company` | string | Yes | Company website or company UUID. Example: `canva.com` or `3fa85f64-5717-4562-b3fc-2c963f66afa6`. |
| `limit` | integer | No | Maximum number of results to return. Default: `10`. Max: `100`. |
| `offset` | integer | No | Number of results to skip. Default: `0`. |

## Response

A successful response returns a status, an array of employee review objects, and pagination metadata.

```json
{
  "status": "success",
  "data": [
    {
      "overview": {
        "rating": 4.1,
        "business_outlook_rating": 0.71,
        "ceo_rating": 0.82,
        "recommend_to_friend_rating": 0.80,
        "compensation_and_benefits_rating": 4.4,
        "culture_and_values_rating": 4.0,
        "diversity_and_inclusion_rating": 4.2,
        "senior_management_rating": 3.6,
        "work_life_balance_rating": 3.9
      }
    }
  ],
  "pagination": {
    "total": 42,
    "limit": 10,
    "offset": 0
  }
}
```

### Top-level response fields

| Field | Type | Description |
| --- | --- | --- |
| `status` | string | Request status. Returns `success` or `error`. |
| `data` | array | Array of employee review objects. |
| `pagination` | object | Pagination metadata for the returned result set. |

### Employee review object fields

| Field | Type | Description |
| --- | --- | --- |
| `overview` | object | Employee review overview metrics. |

### Overview object fields

| Field | Type | Description |
| --- | --- | --- |
| `rating` | float | Overall employee rating on a 0-5 scale. |
| `business_outlook_rating` | float | Percentage positive about business outlook on a 0-1 scale. |
| `ceo_rating` | float | Percentage who approve of the CEO on a 0-1 scale. |
| `recommend_to_friend_rating` | float | Percentage who would recommend the company on a 0-1 scale. |
| `compensation_and_benefits_rating` | float | Compensation and benefits rating on a 0-5 scale. |
| `culture_and_values_rating` | float | Culture and values rating on a 0-5 scale. |
| `diversity_and_inclusion_rating` | float | Diversity and inclusion rating on a 0-5 scale. |
| `senior_management_rating` | float | Senior management rating on a 0-5 scale. |
| `work_life_balance_rating` | float | Work-life balance rating on a 0-5 scale. |

<Note>
  `business_outlook_rating`, `ceo_rating`, and `recommend_to_friend_rating` are returned on a 0-1 scale. All other ratings are returned on a 0-5 scale.
</Note>

### Pagination object fields

| Field | Type | Description |
| --- | --- | --- |
| `total` | integer | Total number of matching results. |
| `limit` | integer | Maximum number of results returned in the response. |
| `offset` | integer | Number of results skipped before the current response. |

## Error responses

### 401 Unauthorized

Returned when the API key is missing or invalid.

```json
{
  "status": "error",
  "error": "UNAUTHORIZED",
  "message": "Invalid or missing x-api-key."
}
```

### 422 Unprocessable Entity

Returned when the required company identifier is missing.

```json
{
  "status": "error",
  "error": "MISSING_PARAMETER",
  "message": "The company field is required."
}
```

### 404 Company Not Found

Returned when no company matches the provided input.

```json
{
  "status": "error",
  "error": "COMPANY_NOT_FOUND",
  "message": "No company found for the provided identifier."
}
```

### 500 Internal Server Error

Returned when an unexpected server error occurs.

```json
{
  "status": "error",
  "error": "SERVER_ERROR",
  "message": "An unexpected error occurred. Please try again."
}
```

## Pagination

Use `limit` and `offset` together to paginate through matching results.

```text
Page 1: limit=10&offset=0
Page 2: limit=10&offset=10
Page 3: limit=10&offset=20
```

<Note>
  Use the `pagination` object to understand the total number of matching results and request the next page.
</Note>

## Implementation notes

- Use the Company Search API first when you need to resolve a company name into a canonical company UUID.
- Pass a website or company UUID in the `company` query parameter.
- Use `limit` and `offset` for pagination.
- Treat `business_outlook_rating`, `ceo_rating`, and `recommend_to_friend_rating` as percentage-style values on a 0-1 scale.
- Treat all other rating fields as 0-5 scale values.
- Store the `overview` object when building company benchmarking, talent, employer brand, or workplace sentiment workflows.
- Do not expose API keys in browser code or public client-side applications.
